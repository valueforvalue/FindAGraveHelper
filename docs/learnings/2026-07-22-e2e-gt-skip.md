# Learning: Why `test_e2e_ground_truth.py` skips the four operator tests

> **Earned by:** Issue #91 (post-pass extraction cleanup). Documents
> the per-test `skipif` gate on the four live e2e tests and the
> smoke variant that replaced the module-level skip.

## What happened

After the post-pass extraction (PRs from Slice 1 onward), the
default suite reported "1 skipped" alongside the integration
"deselect". The skip was at the module level: when
`C:/tmp/ground_truth.csv` is absent, the entire module was
uncollectable, leaving 0 tests in this file and a single skip
line. That masked any regression in the test harness itself.

## Why the operator tests need the external CSV

`test_e2e_ground_truth.py` runs the full FaG searcher against
the operator-downloaded ground-truth CSV (576 records from
dixiedata). Each of the four assertions in the module needs:

- The ground-truth CSV at `C:/tmp/ground_truth.csv`
- A live `playwright-stealth` browser session (L8)
- The 2.5 s throttle floor honored (L1)
- The `test_e2e_searcher_evaluates_with_confusion_matrix`,
  `test_e2e_rank1_hit_rate`, `test_e2e_auto_accept_precision`,
  and `test_e2e_in_top_5` assertions read the live searcher
  output (`results.jsonl` written by `_ensure_searcher_run()`)

All three constraints (L1, L2, L8 from CONTEXT.md) make this
unsuitable for default CI — for the same reason the real-FaG
memory test is excluded (see
`docs/learnings/2026-07-22-real-fag-memory-default-skip.md`).

## What the smoke variant covers

`test_e2e_smoke_evaluation_helpers_with_fixture` (added in this
commit) runs by default with no operator data and no live FaG.
It exercises:

- `compute_confusion_matrix()` with synthesized (actual, score)
  pairs — confirms the matrix math runs.
- `best_threshold()` over the same pairs — confirms the
  threshold search produces a valid `ThresholdResult` (all metrics
  in [0, 1]).

It does NOT assert precision/recall thresholds (those require
the real searcher output). It catches regressions in the
evaluation plumbing itself, without needing any external fixture
beyond the bundled 3-row CSV.

## Operator path to the live e2e tests

```bash
# Download the ground-truth CSV (576 dixiedata records) to:
#   C:/tmp/ground_truth.csv

# Then run:
pytest tests/test_e2e_ground_truth.py -v

# All 5 tests should collect; 4 with full coverage, 1 always-on smoke.
```

The full e2e suite takes ~12 minutes (one searcher run feeds four
assertions via the `_ensure_searcher_run` sentinel).

## Acceptance for future maintainers

The four `@REQUIRES_OPERATOR_GT`-marked tests need the operator
ground-truth CSV AND a live FaG session. The smoke test runs
unconditionally. If you add a fifth operator-gated test, mark it
with `@REQUIRES_OPERATOR_GT` and document the new CSV / FaG
dependency in this file.

The bundled `tests/fixtures/ground_truth_smoke.csv` is a
3-row fixture, NOT the full ground truth. Don't ship the full
CSV in the repo (privacy + size).
